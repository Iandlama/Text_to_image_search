# Vector DB Setup (Qdrant Implementation)

This module handles the infrastructure setup, collection schema configuration, and optimized batch ingestion pipeline for the vector search database using Qdrant (Rust-based vector search engine).

## Architecture & Schema Design (Iteration 1)

For the baseline iteration, the database is configured to support **Multi-Vector Search**, allowing each node/point to store two distinct dense vectors simultaneously to facilitate a hybrid search fusion (Text + Visual).

*   **Collection Name:** `meme_collection_v1`
*   **Vector 1: `vector_image`**
    *   **Model:** `jina-clip-v2` 
    *   **Dimension:** `512` (Truncated)
    *   **Distance Metric:** `Cosine Similarity`
*   **Vector 2: `vector_text`**
    *   **Model:** `multilingual-e5-large` 
    *   **Dimension:** `512` (Truncated)
    *   **Distance Metric:** `Cosine Similarity`

### Payload Schema
To support precise filtering and metadata retrieval for Task 4 (FastAPI backend), each point contains the following payload structures:
*   `id`: Unique identifier (`int` or `UUID`).
*   `ocr_text`: Raw text extracted from the meme image via EasyOCR/Tesseract (Task 1).
*   `file_name`: Reference to the image asset filename (e.g., `<hash>.webp`).

---

## File Structure

*   `init_db.py` — Script to establish connection to Qdrant via gRPC and initialize the collection schema.
*   `upload_data.py` — High-throughput streaming script to ingest `.jsonl` embedding datasets using async-like batching.
*   `search_test.py` — Blueprint and testing utility for multi-vector search execution, designed for FastAPI integration.

## Quick Start & Deployment Guide (Windows / Local)

### 1. Run Qdrant Instance
Ensure Docker Desktop is running on Windows. Start the local Qdrant container with persistent storage enabled:

```powershell
docker run -p 6333:6333 -p 6334:6334 -v C:\qdrant_storage:/qdrant/storage:z qdrant/qdrant
```
*Verify the instance by visiting the Qdrant Dashboard at: `http://localhost:6333/dashboard`*

### 2. Environment Setup
Activate your virtual environment and install the required official python client:
```powershell
# Bypass Windows Execution Policy for the session
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process

# Activate Venv
.venv\Scripts\Activate.ps1

# Install Client
pip install qdrant-client
```

### 3. Initialize Collection Schema
Execute the database initialization script to build the named vector spaces:
```powershell
python init_db.py
```

### 4. Data Ingestion (Awaiting Task 2 Outputs)
Once the `.jsonl` dataset containing the image and text embeddings is generated by the team, place the file in the root directory and trigger the batch uploader:
```powershell
# In upload_data.py, uncomment: upload_jsonl_dataset("final_embeddings.jsonl")
python upload_data.py
```
*Note: Ingestion uses a batch size of 1000 nodes with `wait=False` to optimize hardware overhead and ingestion velocity for the 500,000+ object dataset limit.*
